Semantic elements

HTML markup What is it? Useful tips
<title> The title element is telling us the title of the webpage. The content between the tags is shown in the browser's title bar or tab and will be the default bookmark name if a user adds the page to their favorites. The title will also be shown in as the title for a page in a search engine result. The title element should go in the <head> element in the HTML. It does not go in the <body> of the document.
<header> The <header> element is a container for introductory content such as logo, company name, search feature, headings and navigation links. It usually works as a banner for the page, re-used globally across the website. However, it can also be used within an <article> or a <section> to represent the introduction to that particular piece of content. The <header> usually goes in the top of the page, unless it is nested within an <article> or a <section> - in which case it goes in the top of that particular element.
<nav> The <nav> element is a container for a block of navigation links, for example a menu. It is often nested inside other element such as the <header>. The navigation links can lead to other sections of the same page, to other pages on the same site or to other websites. Not all links must be placed within in a <nav> element!
<main> The <main> element contains the primary content of that particular page. The content should be directly related to the topic of that particular page. The content of the <header> and <footer>, for example, is often more generic and will not be included in the main element. There shouldn’t be more than one <main> element in one page! So unlike the <header> and <footer> element, we don’t have the option to nest it within an <article> or a <section>.
<aside> The <aside> element is used for content that is secondary to the main content of the page. The content is typically separate, but indirectly related to the main content. Examples include additional information, advertisement and navigations. The <aside> element is often used for sidebars!
<footer> The <footer> element holds content that improves the site's usability. It goes at the very bottom of the page and is typically re-used globally across the site. From here the user can usually navigate the site through a site-map, and easily reach things like contact information, social media links, privacy policies, copyright data and information about the author. Similarly to the <header> element, a <footer> element can also be nested within an <article> or a <section>! The main <footer> always goes at the bottom of the page though.
<article> The <article> element is used for content that can stand on it's own - for example a blog post, a newspaper article, a product review or a comment.
<section> The <section> element is used to divide content into logical sub-sections. A <section> element typically groups related content together. An <article> element, for example, could include a few <section> elements, or a <main> element of a page could be divided into a few different <section> elements.
