Welcome!

Do you want to learn the basics of semantic HTML markup? Then you are in the right place!

Let’s get you started by learning what semantic markup is and why it is important - or you might choose to jump straight to a list of some of the most important semantic HTML elements.

What is semantic markup?

Semantic markup is a way of writing HTML that makes it easier to read and understand by humans, assistive technology and search engines. Semantic HTML elements (tags) provide meaning (semantics) to their content. They help us interpret what role the content has and how important it is.

In the early days of the internet, HTML markup was merely used to display scientific documents containing hyperlinks to other documents. As the web grew popular and spread more widely, HTML evolved as well. The need to make the webpages look better grew too, and developers needed easier ways to create more complex layouts. They started to structure their pages through generic elements such as <div>, which allowed them to group content together for styling purposes.

Although a <div> element works fine as a general container for content, it doesn't say anything about the type of content. For reasons explained below, developers needed elements that described what kind of content the markup displayed. This is where semantic markup comes into play. The latest HTML version - HTML5 - includes many useful semantic elements that provide meaning to the content. Check out a few of the most important elements here.

Why is semantic markup important?

There are several good reasons as to why semantic markup is important. Here are the top three:

  1. Accessibility: For sighted users, the layout of a webpage often provides the visual structure needed to identify the different parts of the page. Sighted users can skim through the page and choose what content is relevant for them. Users who are visually impaired or blind should have the same possibility. These users rely on assistive technologies such as screen readers to navigate the page and interact with it's content. Semantic markup helps the technology interpret and communicate the content more accurately, which gives the user a much better experience.
  2. Search Engine Optimization: When it comes to Search Engine Optimization (SEO), webpages displayed through semantic HTML markup will rank higher in search results compared to pages using more generic markup. This is because search engines (just like assistive technologies) rely on semantic markup when interpreting the content and context of a page.
  3. Maintenance: Semantic elements make the HTML markup itself easier to manage. Semantic markup helps the developers to read and understand their own code. When reviewing or editing someone else's code, the consistency provided by the semantics makes the collaboration smoother. If the purpose of each section can be quickly identified, it also becomes easier to update the code. When working with the HTML markup through other languages such as CSS or JavaScript, it is easier to get an idea of the content without having to look too closely in the markup.

Semantic elements

In the table below, you will find a brief explanation of some of the most useful semantic elements. If you want to find more semantic elements and their application, check out the recommended links for further reading.

HTML markup What is it? Useful tips

<title>

 The title element is telling us the title of the webpage. The content between the tags is shown in the browser's title bar or tab and will be the default bookmark name if a user adds the page to their favorites. The title will also be shown as the title for a page in a search engine result. The title element should go in the <head> element in the HTML. It doesn't go in the <body> of the document.

<header>

 The <header> element is a container for introductory content such as logo, company name, search feature, headings and navigation links. It usually works as a banner for the page, re-used globally across the website. However, it can also be used within an <article> or a <section> to represent the introduction to that particular piece of content. The <header> usually goes in the top of the page, unless it is nested within an <article> or a <section> - in which case it goes in the top of that particular element.

<nav>

 The <nav> element is a container for a block of navigation links, for example a menu. It is often nested inside other element such as the <header>. The navigation links can lead to other sections of the same page, to other pages on the same site or to other websites. Not all links must be placed within in a <nav> element!

<main>

 The <main> element contains the primary content of that particular page. The content should be directly related to the topic of that particular page. The content of the <header> and <footer>, for example, is often more generic and will not be included in the main element. There shouldn’t be more than one <main> element in one page! So unlike the <header> and <footer> element, we don’t have the option to nest it within an <article> or a <section>.

<aside>

 The <aside> element is used for content that is secondary to the main content of the page. The content is typically separate, but indirectly related to the main content. Examples include additional information, advertisement and navigations. The <aside> element is often used for sidebars!

<footer>

 The <footer> element holds content that improves the site's usability. It goes at the very bottom of the page and is typically re-used globally across the site. From here the user can usually navigate the site through a site-map, and easily reach things like contact information, social media links, privacy policies, copyright data and information about the author. Similarly to the <header> element, a <footer> element can also be nested within an <article> or a <section>! The primary <footer> always goes at the bottom of the page though.

<article>

 The <article> element is used for content that can stand on it's own - for example a blog post, a newspaper article, a product review or a comment. <article> elements can be nested in <section> elements, and <section> elements can be nested in <article> elements!

<section>

 The <section> element is used to divide content into logical sub-sections. A <section> element typically groups related content together. An <article> element, for example, could include a few <section> elements, or a <main> element of a page could be divided into a few different <section> elements. <article> elements can be nested in <section> elements, and <section> elements can be nested in <article> elements!
